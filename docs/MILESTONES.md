
# DEVELOPMENT MILESTONES

Milestones follow the 12-week execution plan and are structured to always deliver a usable system.

---

## Milestone 1 — Core HTTP & Primitives (Weeks 1–2)

**Goal**: Establish a solid web foundation.

Includes:
- HTTP server integration
- Request abstraction
- Response abstraction

Outcome:
- zypher can accept requests and send structured responses

---

## Milestone 2 — Routing & Middleware (Weeks 3–5)

**Goal**: Deterministic request dispatching.

Includes:
- Static routing
- Typed URL parameters
- Middleware pipeline

Outcome:
- Requests flow cleanly from URL → handler → response

---

## Milestone 3 — Views & Templates (Weeks 6–7)

**Goal**: Server-rendered web pages.

Includes:
- Controller layer
- Template engine
- HTML auto-escaping

Outcome:
- zypher can serve real HTML pages

---

## Milestone 4 — ORM & Migrations (Weeks 8–9)

**Goal**: Persistent data with schema evolution.

Includes:
- SQLite ORM
- Compile-time SQL generation
- Migration system

Outcome:
- Full CRUD applications are possible

---

## Milestone 5 — Forms, Auth & Sessions (Weeks 10–11)

**Goal**: Secure user interaction.

Includes:
- Form handling
- Validation
- Authentication
- Session management

Outcome:
- User-driven applications with security defaults

---

## Milestone 6 — Admin, CLI & Docs (Week 12)

**Goal**: Developer experience and polish.

Includes:
- Auto-generated admin panel
- CLI tooling
- Documentation
- Example project

Outcome:
- zypher is usable by external developers
